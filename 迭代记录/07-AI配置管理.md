## 技术要点

### 密钥安全
- **读取时脱敏**: `GET /api/admin/ai-config` 对含 `api_key`/`secret`/`token` 的 key 自动脱敏（`sk-1****abcd`），仅显示首 4 位 + 末 4 位
- **仅管理员访问**: 所有 `/api/admin/ai-config` 路由受 `authMiddleware` 保护（JWT 校验）
- **存储于服务端**: API Key 存于 SQLite `ai_config` 表，不对前端暴露完整明文

### 优先级设计
书灵对话获取 API Key 优先级: 数据库 `ai_config` 表 > 环境变量 `DEEPSEEK_API_KEY`。这使得后台配置可覆盖环境变量默认值。

### 前端交互
- Key 输入框默认 `type="password"`，提供显示/隐藏切换
- 保存时分别调用 `setAiConfig('deepseek_api_key', ...)` 和 `setAiConfig('deepseek_model', ...)`，二者独立请求
- 保存成功后显示"已保存"提示，2 秒后自动消失
- 页面加载时自动检测当前配置状态（已配置/未配置/检测中）
