## 技术要点

### 1. localhost:3000 硬编码问题

5 个 Vue 组件共 7 处将图片 URL 拼接为 `'http://localhost:3000' + path`：

- `AboutView.vue` — 头像 `avatarStyle`
- `BlogView.vue` — 头像 `avatarStyle` + `coverImg()`
- `BlogPostView.vue` — 头像 `avatarStyle` + `coverImg()`
- `GalleryView.vue` — 画作展示 `currentImg`
- `AdminGallery.vue` — 图片列表 `imgUrl()`

开发环境中 Vite 代理将 `/uploads/` 转发到 API 服务，所以能正常工作。生产环境浏览器从 `http://amadeuscristina.site` 发起对 `http://localhost:3000` 的请求时，Chrome 的 Private Network Access 策略直接拦截（loopback 地址属于 more-private address space）。

**修复**：上传接口返回的 `url` 已经是 `/uploads/...` 格式，Nginx 配置了 `/uploads/` 代理到 Express，所以直接去掉前缀拼接即可。

```
// Before
return url.startsWith('http') ? url : 'http://localhost:3000' + url;

// After
return url.startsWith('http') ? url : url;
```

### 2. Nginx proxy_pass 尾部斜杠

Nginx `proxy_pass` 末尾有无 `/` 决定是否剥离匹配的前缀：

| 配置 | 请求 `/api/music` | 转发到 Express |
|------|-------------------|----------------|
| `proxy_pass http://127.0.0.1:3000;` | → | `/api/music` ✓ |
| `proxy_pass http://127.0.0.1:3000/;` | → | `/music` ✗ |

Express 中所有路由均以 `/api/` 前缀挂载（如 `app.use('/api/music', musicRouter)`），因此 **不能** 加尾部斜杠。之前为了修复 `/health` 404 而错误地加了 `/`，导致所有业务接口 404，正确做法是将 `/health` 改为 `/api/health`。

### 3. Nginx location 优先级

Nginx location 匹配优先级（从高到低）：

1. `location = /exact` — 精确匹配
2. `location ^~ /prefix` — 优先前缀匹配
3. `location ~ \.regex$` — 正则匹配
4. `location /prefix` — 普通前缀匹配

服务器配置中 `location ~ .*\.(png|jpg|gif)$` 是正则匹配，优先级高于普通 `location /uploads/`，导致 `/uploads/xxx.png` 被静态文件规则拦截，无法代理到 Express。加 `^~` 提升为优先前缀匹配即可。

### 4. 网易云音乐 stream 代理

`GET /api/music/:id/stream` 端点：
1. 从数据库查询歌曲记录，提取 `netease_id`
2. 用 cookie 调用 NetEase `song_url` API 获取临时 mp3 地址
3. 通过 `fetch()` 代理音频流，以 206 Range 请求支持拖动进度条

前端 `MusicPlayer.vue` 中 NetEase 歌曲自动路由到 `/api/music/:id/stream`，播放失败时也自动重试代理端点。
